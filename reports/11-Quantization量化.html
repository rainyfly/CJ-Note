11 - Quantization 量化

SGLang 量化方案全景：FP8、INT8、GPTQ、AWQ 及更多

1. 量化概述

SGLang 在 srt/layers/quantization/ 下实现了 20+ 种量化方案，覆盖权重量化 (Weight-Only) 和权重-激活量化 (Weight-Activation) 两大类。量化系统通过插件化设计与 Linear 层解耦。

2. 量化架构

flowchart TD A["ServerArgs.quantization"] --> B["QuantizationConfig 工厂"] B --> C{"量化方法选择"} C --> D["FP8Config"] C --> E["GPTQConfig"] C --> F["AWQConfig"] C --> G["Int8Config"] C --> H["BitsAndBytesConfig"] C --> I["MarlinConfig"] C --> J["...其他"] D --> K["get_quant_method(layer)"] K --> L["QuantizedLinearMethod"] L --> M["替换 Linear.forward()"] L --> N["替换 Linear.weight 参数创建"]

量化系统核心接口

class QuantizationConfig:
    """量化配置基类"""
    @abstractmethod
    def get_quant_method(self, layer, prefix) -> QuantizeMethodBase:
        """返回该层对应的量化方法"""

class QuantizeMethodBase:
    """量化方法基类"""
    def create_weights(self, ...):  # 创建量化权重参数
    def apply(self, ...):           # 量化前向计算

4. INT8 量化

W8A8

w8a8_int8.py + int8_kernel.py 实现 INT8 权重-激活量化：

  • Per-token 动态激活量化
  • Per-channel 静态权重量化
  • SmoothQuant 风格的 INT8 GEMM

5. GPTQ

Weight-Only 4-bit / 8-bit

gptq.py 支持 GPTQ 量化模型的推理：

  • 支持 4-bit 和 8-bit 权重
  • 支持 group_size 分组量化
  • 底层使用 CUDA kernel 进行反量化+GEMM
  • 可选 Marlin kernel 加速 (4-bit)

6. AWQ

Weight-Only 4-bit

awq.py + awq_triton.py 支持 AWQ (Activation-aware Weight Quantization)：

  • 4-bit 权重量化，保护重要通道
  • Triton 反量化 kernel
  • 可切换 Marlin 后端获得更高性能

7. Marlin

高性能 4-bit

marlin_utils.py + marlin_utils_fp8.py 集成了 Marlin 高性能量化 kernel：

  • 针对 4-bit Weight-Only 量化的高度优化 CUDA kernel
  • 支持 GPTQ 和 AWQ 格式的 Marlin 后端
  • FP8 Marlin 变体
  • 接近 FP16 的推理速度

8. BitsAndBytes

易用 4-bit / 8-bit

bitsandbytes.py 集成了 BitsAndBytes 库：

  • NF4 (NormalFloat4) 量化
  • LLM.int8() 混合精度
  • 简单易用，适合快速部署

9. MXFP4

前沿 4-bit

mxfp4.py + mxfp4_tensor.py 实现了 Microscaling FP4 格式：

  • 基于 OCP MX 标准的 FP4 格式
  • Block-wise 缩放 (block_size=32)
  • 适用于下一代硬件加速

10. KV Cache 量化

kv_cache.py + kvfp4_tensor.py 实现了 KV Cache 的量化压缩：

KV Cache FP8

推荐

将 KV Cache 从 FP16 压缩到 FP8，显存节省约 50%，精度损失极小。

KV Cache FP4

实验性

进一步压缩到 FP4，显存节省 75%，适用于超长上下文场景。

11. Compressed Tensors 框架

compressed_tensors/ 子目录实现了 Neural Magic 的 Compressed Tensors 格式支持：

  • 统一的量化模型加载格式
  • 支持多种 scheme (W8A8-INT8, W8A8-FP8, W4A16, etc.)
  • 与 SparseML/DeepSparse 生态兼容
