3.1 分布式模型加载

  • 权重分片：权重按 TP 维度切分，每个 rank 仅加载对应分片
  • 不均匀 CPU TP：支持在 CPU 上用不同数量的 TP 加载然后调整到目标 TP
  • 慢 rank 检测：通过 slow_rank_detector 监测加载不均衡
  • 远程权重加载：支持通过 transfer engine 从远程实例加载权重

7. CUDA Graph 机制

CUDA Graph 通过预录制 GPU kernel 调用图来消除 CPU launch 开销，是 decode 阶段的关键优化。

核心思想：在初始化阶段，针对每个可能的 batch size 预先捕获一次完整的前向计算图。运行时只需更新输入缓冲区并回放图，避免逐 kernel 的 CPU dispatch 开销。

8. 图捕获流程

sequenceDiagram participant CGR as CudaGraphRunner participant GPU as GPU participant AB as AttentionBackend participant Model as Model CGR->>CGR: capture() 开始 loop 对每个 BS (从大到小) CGR->>CGR: 填充输入缓冲区(dummy data) CGR->>AB: init_forward_metadata_capture_cuda_graph() CGR->>CGR: warmup 前向 (不录制) CGR->>GPU: torch.cuda.CUDAGraph() GPU->>GPU: graph.capture() 开始 CGR->>Model: forward(input_ids, positions, ...) Model-->>CGR: output GPU->>GPU: graph.capture() 结束 CGR->>CGR: 保存 graph 和 output_buffer end

关键实现细节

  • 倒序捕获：从最大 BS 到最小 BS 捕获，利于内存在不同图之间共享
  • 全局图内存池：global_graph_memory_pool 在所有 CudaGraphRunner 间共享
  • GC 冻结：捕获期间冻结 GC 避免捕获到垃圾回收操作
  • torch.compile 集成：对小于 torch_compile_max_bs 的 BS 使用 compiled forward
  • PDMux 支持：PD multiplexing 模式下每个 stream 独立捕获

12. 总结

ModelExecutor 关键设计要点

  • 三层批次抽象：ScheduleBatch(CPU) -> ModelWorkerBatch(过渡) -> ForwardBatch(GPU)，清晰分离调度逻辑和计算逻辑
  • CUDA Graph 加速：对 decode 阶段的小批次计算，通过图捕获消除 CPU launch 开销
  • torch.compile 结合：在 CUDA Graph 中嵌入编译优化的 kernel，进一步减少计算开销
  • 灵活的注意力后端：通过注册表模式支持 10+ 种注意力后端，自动选择最优实现
  • 内存精细管理：多种 KV 缓存池类型适配不同模型架构(MHA/MLA/SWA/Mamba)
  • 投机解码原生集成：CUDA Graph 对 verify 模式的原生支持
  • Padding 优化：当 disable_padding=False 时自动向上对齐 BS，减少图数量
  • PP 支持：通过 PPProxyTensors 在流水线阶段间传递中间张量
