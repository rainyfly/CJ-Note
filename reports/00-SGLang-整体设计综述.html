SGLang 整体设计综述

SGLang Runtime (SRT) — 高性能 LLM 推理引擎架构全景分析

1. 项目概述

SGLang 是一个由 UC Berkeley LMSYS 团队主导开发的高性能大语言模型推理与服务框架。项目的名称源自 Structured Generation Language，即"结构化生成语言"。它包含两大核心组件：

SGLang Runtime (SRT)

高性能推理后端引擎，负责模型加载、请求调度、KV Cache 管理、模型前向计算等。提供 OpenAI 兼容的 HTTP API。

Inference Engine CUDA Optimized

SGLang Frontend (Lang)

结构化生成的前端 DSL，允许开发者以 Python 函数的方式定义复杂的 LLM 交互流程，支持 fork/join、选择、约束生成等。

2.2 核心进程模型

SGLang 采用多进程架构，通过 ZMQ (ZeroMQ) 进行进程间通信：

进程角色关键类说明
主进程 HTTP Server + TokenizerManager TokenizerManager 接收 HTTP 请求，执行分词/反分词，通过 ZMQ 与 Scheduler 通信
Scheduler 子进程 请求调度 + 模型执行 Scheduler, ModelRunner 核心调度引擎，管理 prefill/decode 批次，驱动模型前向计算
Detokenizer 子进程 反分词 DetokenizerManager 将 token IDs 转换回文本，支持流式输出
DP Controller 数据并行控制 DataParallelController 多 Scheduler 实例间的负载均衡 (dp_size > 1 时启用)

5. 核心功能特性

RadixTree 前缀缓存

核心创新

基于 Radix Tree 实现高效的 KV Cache 前缀复用。相同前缀的请求可以共享已计算的 KV Cache，大幅减少 prefill 计算量。支持 LRU/LFU 等多种淘汰策略。

Continuous Batching

调度优化

连续批处理调度，支持 prefill 和 decode 请求的动态混合调度。通过 chunked prefill 控制首 token 延迟。

CUDA Graph

性能优化

对 decode 阶段的前向计算进行 CUDA Graph 捕获，消除 CPU 调度开销，显著提升 decode 吞吐量。

投机解码 (EAGLE)

推测加速

支持 EAGLE/EAGLE-2/EAGLE-3 和 N-Gram 投机解码算法，通过 draft model 预测多个 token 候选，降低 decode 延迟。

约束解码 / 结构化输出

生成控制

支持 XGrammar、Outlines、LLGuidance 多种 Grammar 后端，实现 JSON Schema、正则表达式等约束生成。

MoE 优化

效率优化

Fused MoE Triton kernel、Expert Parallelism (EP)、Token Dispatcher、EPLB (Expert Parallel Load Balancing) 等 MoE 特化优化。

多量化格式

压缩推理

FP8 (W8A8)、INT8、GPTQ、AWQ、Marlin、BitsAndBytes、MXFP4、KV Cache FP8 等 20+ 种量化方案。

Prefill-Decode 分离

架构创新

支持 P/D 分离部署，prefill 和 decode 阶段在不同机器上执行。通过 NIXL/Mooncake 进行 KV Cache 传输。

7. 模型支持

SGLang 支持 100+ 种模型架构，通过 srt/models/registry.py 管理模型注册。主要分类如下：

Dense LLM

LLaMA 1/2/3/4, Qwen 1/2/3, Mistral, DeepSeek, GLM-4, GPT-2, OPT, Phi-1/2/3/4, Gemma 1/2/3, InternLM, ChatGLM, XVERSE, Baichuan, StableLM, Command-R, DBRX, Falcon, Granite, Olmo, Exaone, etc.

MoE LLM

Mixtral, Qwen2-MoE, Qwen3-MoE, DeepSeek-V2/V3, GLM4-MoE, XVERSE-MoE, OLMoE, GraniteMoE, PhiMoE, Grok, AF-MoE, Bailing-MoE, Hunyuan, etc.

Vision-Language

LLaVA, InternVL, Qwen2-VL, Qwen2.5-VL, Qwen3-VL, GLM-4V, MiniCPM-V, Pixtral, MLLaMA, Phi-4-MM, NVILA, DeepSeek-Janus, Kimi-VL, etc.

Embedding & Classification

LLaMA-Embedding, LLaMA-Classification, Qwen2-Classification, BERT, RoBERTa, InternLM2-Reward, Gemma2-Reward, etc.
