2.1 注册机制

ATTENTION_BACKENDS = {}

def register_attention_backend(name):
    def decorator(fn):
        ATTENTION_BACKENDS[name] = fn
        return fn
    return decorator

@register_attention_backend("flashinfer")
def create_flashinfer_backend(runner):
    if not runner.use_mla_backend:
        return FlashInferAttnBackend(runner)
    else:
        return FlashInferMLAAttnBackend(runner)

3. 基类设计 (AttentionBackend)

AttentionBackend 是所有注意力后端的抽象基类，定义了统一接口。

3.2 forward 分发逻辑

def forward(self, q, k, v, layer, forward_batch, save_kv_cache=True):
    if forward_batch.forward_mode.is_idle():
        return q.new_empty(q.shape[0], layer.tp_q_head_num * layer.v_head_dim)
    elif forward_batch.forward_mode.is_decode():
        return self.forward_decode(q, k, v, layer, forward_batch, ...)
    else:
        return self.forward_extend(q, k, v, layer, forward_batch, ...)

5.4 Tensor Core Decode 优化

decode_use_tensor_cores：当 KV 缓存使用 FP8 数据类型或 Q 头数 / KV 头数比 > 4 时，FlashInfer 自动切换到 Tensor Core 实现以获得更高性能。

7. FlashAttention 后端

支持 FlashAttention v3 (SM80+) 和 v4 (SM90+)，通过 sgl_kernel 调用。

7.1 FlashAttentionMetadata

字段描述
cache_seqlens_int32KV 缓存序列长度
cu_seqlens_q / cu_seqlens_kQuery/Key 的累积序列长度
max_seq_len_q / max_seq_len_k最大 Q/K 序列长度
page_table分页 KV 缓存的页表
window_size滑动窗口大小元组

7.2 Local Attention 支持

FlashAttention 后端支持 local attention（如 Gemma 的滑动窗口），通过 LocalAttentionMetadata 将长序列分割成虚拟 batch 项，每个 chunk 独立处理。

Local Attention 分块策略：对于 attn_chunk_size=4 的序列，q_seqlens=10 的请求会被分成多个虚拟 batch 项：每个项只关注自己 chunk 内和之前所有 chunk 的 KV。这在保持因果性的同时限制了每次注意力的范围。

  • Logits 处理时正确切片各 rank 的输出
  • CUDA Graph 的 can_run 判断

12. 总结

SGLang 注意力系统的核心特点

  • 注册表模式：15+ 种后端通过装饰器统一注册，运行时按名称选择
  • 统一抽象接口：AttentionBackend 基类定义了 forward_decode/forward_extend/init_metadata 等标准接口
  • FlashInfer 默认后端：支持 paged KV、ragged prefill、Tensor Core decode、FP8 等特性
  • Triton 可定制后端：自定义 kernel 支持 KV split、double sparsity 等高级特性
  • MLA 专用优化：FlashMLA/FlashInfer-MLA/Cutlass-MLA 针对 DeepSeek MLA 架构深度优化
  • 分页 KV 管理：CSR 格式的索引结构支持非连续内存的高效访问
  • DP Attention：在数据并行模式下分割注意力计算，仅在 MLP 层做 AllReduce
  • 混合架构支持：透明包装不同后端以支持 Mamba/GDN/KimiLinear 等混合模型
  • CUDA Graph 集成：每个后端实现图捕获/回放所需的元数据初始化接口
