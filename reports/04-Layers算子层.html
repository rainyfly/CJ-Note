2. Linear 线性层

linear.py 实现了支持张量并行的线性层，是模型中最基础也是计算密集度最高的组件。

线性层类型

类并行模式说明
ReplicatedLinear复制每个 TP rank 都有完整权重
ColumnParallelLinear列并行按列切分 weight，AllGather 输出
RowParallelLinear行并行按行切分 weight，AllReduce 输出
QKVParallelLinear列并行QKV 投影合并，head 维度切分
MergedColumnParallelLinear列并行多个列并行层合并 (gate+up)

2.1 量化集成

Linear 层与量化系统深度集成。每个线性层创建时会检查当前量化配置，通过 quant_config.get_quant_method() 获取对应的量化方法，替换默认的 weight 和 forward 实现。

4. LayerNorm / RMSNorm

layernorm.py 实现了高性能的归一化层：

  • RMSNorm: Root Mean Square Normalization，LLaMA 系列使用
  • GemmaRMSNorm: Gemma 变体 (+1 偏移)
  • LayerNorm: 标准 Layer Normalization

底层调用 FlashInfer 或自定义 CUDA kernel 实现高效计算，支持 FP16/BF16 输入。

5. 激活函数

activation.py 提供了 fused 激活函数实现：

  • SiluAndMul: SiLU(x) * y，用于 LLaMA 风格的 gate-up projection
  • GeluAndMul: GELU(x) * y
  • ScaledActivation: 带缩放因子的激活

这些 fused 实现将激活函数和逐元素乘法合并为单个 kernel，减少显存读写。

7. Logits Processor

logits_processor.py 负责将模型最后一层的 hidden states 转换为 logits：

处理流程

  1. 从 hidden states 中提取目标位置（最后一个 token 或所有 token）
  2. 与 LM Head 权重做矩阵乘法得到 logits
  3. 应用 logit softcapping（如 Gemma2）
  4. 根据需要收集 logprobs
  5. 应用 Grammar-based 掩码（约束解码）
  6. 应用 penalty（频率/存在/重复惩罚）
  7. 执行采样

8. Embedding 层

vocab_parallel_embedding.py 实现了词汇表并行的嵌入层：

  • VocabParallelEmbedding: 按词汇表维度切分到各 TP rank
  • ParallelLMHead: 并行 LM Head，与 Embedding 共享权重

9. DP Attention

dp_attention.py 实现了 Data Parallel Attention 特性。在 DP 模式下，不同数据并行 rank 的请求需要在 MLP 层做 AllReduce 同步，但 Attention 部分各自独立计算。这是 SGLang 的核心性能优化之一。

10. Communicator 通信层

communicator.py 封装了 TP 通信操作，提供统一接口：

  • all_reduce(): 全规约
  • all_gather(): 全收集
  • reduce_scatter(): 规约散射

底层支持 NCCL、PyNCCL、Custom AllReduce (基于共享内存) 等多种通信后端。
