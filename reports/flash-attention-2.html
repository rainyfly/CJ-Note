Flash Attention 2 论文研读

TYPE: PAPER READING DATE: 2026-04-10 TOPIC: Attention Kernel / GPU Parallelism

TL;DR

  • FA2 通过减少非 matmul 运算量、提升 GPU 线程利用率，在 A100 上实现了约 2x 的相对加速（相比 FA1），达到理论 FLOPS 利用率的 72%。
  • 核心算法改变：将序列维度也作为并行维度，每个 thread block 负责独立的 Query 分块，彻底消除了 FA1 中跨 warp 的 softmax 归一化通信开销。
  • 工作划分从"列优先"改为"行优先"——外循环遍历 Q 分块，内循环遍历 K/V 分块，使每个 warp 可以独立完成归一化，无需 warp 间同步。
  • 专门针对 causal masking 做了 skipping 优化：整块不在对角线附近的 K/V 分块直接跳过，减少约 50% 的 causal attention 计算量。
  • 原生支持 Multi-Query Attention (MQA) 和 Grouped-Query Attention (GQA)，以及头维度最大 256 的更宽 head dim。

1. 背景：为什么还需要 Flash Attention 2

Transformer 的 Self-Attention 计算复杂度是序列长度 N 的 O(N²)，其中最大瓶颈是将 N×N 大小的注意力矩阵读写到 GPU HBM（高带宽显存）。Flash Attention 1（2022年，Dao et al., NeurIPS 2022）通过 Tiling 技术将整个 softmax + matmul 融合成单个 CUDA kernel，将注意力矩阵访问限制在 SRAM 内，从而大幅减少 HBM 读写，降低内存占用从 O(N²) 到 O(N)。

FA1 已经相比标准实现快 2-4x，但实际测量发现：在 A100 GPU 上，FA1 只能达到理论 FLOPS 利用率的约 30-35%，而理论上 GPU 的 matmul 效率可以做到 70%+ 。原因在于 FA1 有多处非 matmul 计算开销和不合理的线程工作划分，这就是 FA2 要解决的问题。

核心矛盾 现代 GPU（如 A100）的 Tensor Core FP16 matmul 吞吐量极高（312 TFLOPS），但 CUDA Core 的标量/向量运算速度慢得多（约 20 TFLOPS）。任何注意力实现中的非 matmul 运算（rescaling、softmax 指数等）都是严重的性能瓶颈，必须被极度压缩。

2. Flash Attention 1 回顾与局限

FA1 的核心思路

FA1 的算法框架：将 Q、K、V 分成大小为 B_r × d 和 B_c × d 的块（block），分别放入 SRAM。对每个 Q 块，遍历所有 K/V 块，在 SRAM 内计算局部 softmax，使用 online softmax trick 动态更新归一化因子 m（行最大值）和 l（行 exp sum），最终得到正确输出——无需将完整 N×N 矩阵写回 HBM。

FA1 的三个关键局限
  1. 大量非 matmul 运算：FA1 在每个 K/V 块迭代时需要频繁 rescale 中间输出，这些 rescale 操作是逐元素的标量乘法，由 CUDA Core 完成，拖慢整体吞吐。
  2. 线程块工作划分低效：FA1 以 batch × heads 作为并行维度，每个 thread block 处理一个 head。当 batch size 小、head 数量不多时（如推理阶段），这会导致 GPU SM 利用率不足。
  3. Warp 间通信开销：FA1 在每个 thread block 内，不同 warp 处理 Q 矩阵的不同列（K 维度），但 softmax 归一化需要对整行求和，导致 warp 间需要通过 shared memory 进行归约通信（all-reduce），产生同步开销。

3. FA2 核心算法改进

3.1 减少非 matmul 运算量

FA1 在前向传播中，每次处理新的 K/V 块时需要对已经累积的输出矩阵做 rescale：

// FA1 style — 每次内循环都要 rescale
O_new = diag(exp(m_old - m_new)) * O_old + exp(S_new - m_new) * V_new
l_new = exp(m_old - m_new) * l_old + rowsum(exp(S_new - m_new))

FA2 的改进：延迟归一化（deferred rescaling）——在整个 K/V 块迭代完成之前，不对累积输出 O 进行归一化，只在最后一次迭代后做一次归一化。中间迭代只更新 m 和 l 的统计量，而不立即 rescale 输出矩阵。

// FA2 style — 内循环中不 rescale O，只更新统计量
for each K/V block j:
    S_j = Q * K_j^T                    // matmul (Tensor Core)
    m_j = max(m_{j-1}, rowmax(S_j))    // 更新行最大值
    P_j = exp(S_j - m_j)               // 局部 softmax 分子
    l_j = exp(m_{j-1} - m_j) * l_{j-1} + rowsum(P_j)  // 更新 sum
    O_j = exp(m_{j-1} - m_j) * O_{j-1} + P_j * V_j   // 累积（仍需 rescale）

// 最终归一化：只做一次
O = O_K / l_K

实际上 FA2 进一步优化：在中间累积时不乘以 1/l，保存未归一化的 O，仅在最后输出时做一次除法。这样将每个 K/V 块迭代中的 rescale 运算减少到最小，主要让 Tensor Core 做 matmul。

为什么 rescale 减少很重要 在 A100 上，FP16 matmul 的 Tensor Core 吞吐量是 FP32 CUDA Core 的约 16 倍。FA1 的大量 rescale 和 elementwise 操作由 CUDA Core 完成，严重拖慢了整体速度。FA2 将这些运算减少到接近最低限度，使 Tensor Core 利用率显著提升。

3.2 循环顺序交换（外循环换成 Q 块）

这是 FA2 最关键的算法改变。FA1 的循环结构是：

// FA1: 外循环 Q 块，内循环 K/V 块（但 warp 级别按 K 维切分）
for each Q block i:          // outer loop → thread block
    for each KV block j:     // inner loop
        compute partial S, update m, l, O

FA1 中，一个 thread block 内的多个 warp 分工处理同一 Q 行的不同 K 列，最终需要在 warp 之间做 softmax 归约，有跨 warp 同步开销。

FA2 改变了 warp 级别的工作切分方式：

FA1 Warp 划分

每个 warp 负责 K/V 的不同列块，多个 warp 协作计算同一 Q 行的注意力分数。需要 warp 间 all-reduce 归一化统计量。

通信：warp 间 shared memory 同步

FA2 Warp 划分

每个 warp 负责 Q 的不同行块，每个 warp 独立遍历所有 K/V 块，独立完成自己行的 softmax 归一化。

通信：完全无跨 warp 同步

4. GPU 并行性提升

4.1 序列维度并行（Parallelism over Sequence Length）

FA1 的并行维度是 batch_size × num_heads：每个 thread block 处理一个 (batch_idx, head_idx) 对。当 batch size = 1、heads = 32 时，只能并发运行 32 个 thread block，而 A100 有 108 个 SM，大量 SM 会处于空闲状态——这在推理阶段（小 batch）是严重问题。

FA2 增加了序列维度的并行：并行维度变为 batch_size × num_heads × ceil(N/B_r)，其中 B_r 是 Q 分块大小。每个 thread block 只处理 Q 序列的一个子范围，多个 thread block 可以并发处理同一个 head 的不同 Q 行。

这对于长序列尤其关键：序列长度 4096、heads 32 时，FA2 可以并发 32 × (4096/64) = 2048 个 thread block，远超 A100 的 SM 数量，GPU 利用率趋近于 100%。

为什么 FA1 无法简单地做序列并行 FA1 在每个 thread block 内，warp 按 K 维度切分，因此不同 warp 持有同一 Q 行的不同部分注意力分数，必须在块内汇总。这导致一个 Q 行必须由同一 thread block 完整处理，无法将序列维度进一步切开分配给不同 thread block。FA2 的 warp 按 Q 行切分之后，每个 Q 行完全属于一个 warp，不同 Q 行可以完全独立地给不同 thread block 处理。

4.2 Forward 与 Backward 的分开并行

在反向传播中，FA2 同样做了序列维度并行的优化。FA1 的 backward pass 中，dQ 的计算需要对所有 K/V 块的贡献求和，这本质上是一个规约操作，FA1 用了一次 atomic add 写回 HBM，会产生写冲突。FA2 通过让每个 thread block 只处理一段 K/V，最后做一次干净的规约来避免 atomic 竞争，backward 速度也提升了约 2x。

4.3 Warp 内部的 matmul 利用率

FA2 精心设计了 Q、K、V 块的大小（B_r、B_c）和 head dimension d，使得内层的 matmul 操作恰好对齐 Tensor Core 的最优 tile size（一般是 16×16 或 16×8 的 wmma 指令）。FA2 支持 head dim 最大 256（FA1 最大 128），对更宽的 head 同样能维持高 Tensor Core 利用率。

5. 工程实现关键细节

5.1 Causal Masking 的 Skip 优化

在 causal（自回归）注意力中，位置 i 的 Q 只能 attend 到位置 ≤ i 的 K/V。对于 Q 分块 [i_start, i_end] 和 K/V 分块 [j_start, j_end]：

  • 如果 j_end ≤ i_start（K/V 块完全在 Q 块左侧）：整块都是有效的 attend，无需任何 masking，直接做矩阵乘。
  • 如果 j_start > i_end（K/V 块完全在 Q 块右侧）：整块都是 masked out，可以完全跳过。
  • 只有对角线附近的块才需要逐元素地应用 causal mask。

FA2 在 kernel 内对这三种情况做了显式区分。对于 causal attention，右上角的 K/V 块（masked 区域）直接 skip，有效运算量减少约 50%（与非 causal attention 相比）。

// Pseudocode: FA2 causal skip logic
for j in range(num_kv_blocks):
    j_start = j * B_c
    j_end   = j_start + B_c

    if j_start > i_end:
        break  // 后续所有 K/V 块均 masked，直接跳出

    if j_end <= i_start:
        // 整块无 mask，直接做 matmul，无需逐元素 masking
        S = Q_block @ K_block.T
    else:
        // 边界块：需要逐元素应用 causal mask
        S = Q_block @ K_block.T
        S = apply_causal_mask(S, i_start, j_start)
实际性能影响 在 GPT 风格的预填充（prefill）阶段，序列长度越长，右上角的 skip 区域越大（占所有块的约 50%）。FA2 的 causal 模式几乎与非 causal 的一半 FLOP 量完美对应，避免了对无效区域的任何计算。

5.2 Shared Memory 布局与 Bank Conflict 消除

FA2 的 SRAM（shared memory）布局针对 Ampere/Hopper 架构的 128-byte cache line 做了对齐。Q、K、V 块在 shared memory 中的 stride 被精心设置，使得 warp 内 32 个线程的访问模式恰好 coalesce，不产生 bank conflict。

具体地，对于 head dim = 64 的 FP16，每行 128 bytes 恰好是一个 cache line，每个 warp 一次加载 16 行（256 字节/线程），满足 128-byte 对齐要求。head dim = 128 时，FA2 在 shared memory 中使用 swizzled layout，错开相邻行的 bank offset，消除 conflict。

5.3 Register 压力与 Occupancy 的权衡

Flash Attention 的 SRAM 需求量决定了每个 SM 能运行多少个 thread block（occupancy）。FA2 通过以下方式优化：

参数 FA1 FA2 影响
Q block 行数 B_r 64 64 (forward) / 32 (backward) 控制 SRAM 占用
K/V block 列数 B_c 64 64 (forward) / 64 (backward) 内循环 tile 大小
head dim 上限 128 256 支持更大 head
Warp 数量/TB 4 4 (forward) / 8 (backward) backward 用更多 warp 摊薄 latency

FA2 的 backward pass 特别重要：FA1 的 backward 有时会因为寄存器压力过高导致 register spilling（溢出到 L2 缓存），FA2 通过精确调整 tile size 和寄存器分配，避免了 spilling，backward 速度提升尤为显著。

5.4 Multi-Query Attention (MQA) 与 Grouped-Query Attention (GQA)

MQA（Shazeer 2019）和 GQA（Ainslie et al. 2023）中，K/V heads 数量远少于 Q heads（如 Q 有 32 个 head，K/V 只有 1 或 8 个）。FA2 原生支持这种模式：

// GQA 中 K/V 的 head index 计算
kv_head_idx = q_head_idx // (num_q_heads // num_kv_heads)

// 不同 Q head 共享同一 K/V head 的内存地址
K_ptr = K_base + kv_head_idx * kv_head_stride
V_ptr = V_base + kv_head_idx * kv_head_stride

这样在 kernel 内无需额外的 K/V 广播，多个 Q head 的 thread block 直接读取同一 K/V 地址，只是通过索引映射来完成，不产生额外 HBM 流量。

5.5 前向传播保存 softmax 统计量用于反向传播

标准注意力的反向传播需要前向传播的中间激活值（整个 N×N 注意力矩阵 P）。Flash Attention 系列的核心创新是：只需保存 m（行最大值）和 l（行 exp sum），就能在反向传播时重新计算所需的局部注意力分数，节省 O(N²) 的内存，仅需 O(N)。

FA2 将 m 和 l 合并为一个向量 L = m + log(l)（log-sum-exp 值），这样反向传播中只需一个标量即可重建 softmax 分子，进一步减少前向传播需要存储的数据量。

// FA2 forward: 保存 LSE (log-sum-exp)
L[i] = m_i + log(l_i)   // 每行一个标量，O(N) 存储

// FA2 backward: 用 L 重建注意力权重
// 对于 (Q_i, K_j) 对:
S_ij = Q_i @ K_j^T / sqrt(d)
P_ij = exp(S_ij - L[i])   // 直接得到归一化后的注意力权重

Backward Pass 的提升更显著 FA2 相比 FA1 在 backward pass 上的提升（约 2.5x）大于 forward pass（约 2x）。这是因为 backward 本来就有更多非 matmul 运算和 warp 间通信，FA2 对这些的改进尤为明显。在实际训练中，backward 通常占总计算时间的 2/3，因此 backward 的加速对训练吞吐量影响更大。

推理阶段（小 batch）的提升

在自回归生成（decode）阶段，batch size 通常为 1-8，序列在不断增长。FA1 此时的 GPU 利用率极低（<10%），因为并行度只有 batch × heads，远不够填满 GPU。FA2 加入序列维度并行后，即使 batch=1，依然能通过序列分块给多个 SM 分配工作，利用率显著提升。

7. 局限与后续工作

7.1 局限

  • CUDA 专属：FA2 的实现依赖 CUDA 的 warp-level 原语（__shfl_sync、wmma 指令），不支持 ROCm/AMD GPU 或其他加速器（TPU、IPU）。
  • 序列长度对 BlockSize 的敏感性：block size 选择（B_r、B_c）在不同 head dim 和序列长度下有所不同，论文给出的最优 block size 并不是通用的，需要 autotuning。
  • 不支持稀疏注意力模式：FA2 假设全 dense attention（或标准 causal attention），对于 sliding window、Longformer 类稀疏 pattern 需要额外适配。
  • Decode 阶段依然受限：在 batch=1 的逐 token decode 时，seq_q = 1，序列并行无法帮助，仍然是 memory-bound 的操作，此时 FA2 的优势不如 prefill 阶段明显。
  • Dropout 支持限制：FA2 的 attention dropout 实现需要保存随机数状态，在很长序列下有额外开销。

7.2 后续工作（Flash Attention 3）

FA3（2024年，针对 Hopper H100 架构）进一步：利用 H100 的异步 TMA（Tensor Memory Accelerator）做 overlap 加载与计算、利用 wgmma 指令做更大粒度的 warp group matmul、支持 FP8 精度。FA3 在 H100 上达到了约 75% 的 FP16 FLOPS 利用率（H100 FP8 可达 ~85%）。

8. Takeaways

对 LLM 从业者的意义

  • 训练加速：FA2 使得在单 A100 上训练更长序列（4K、8K、32K）成为可能，而不需要增加显存。GPT-4 规模的模型训练中，attention kernel 的时间占比可能高达 30-40%，FA2 直接将这部分时间减半。
  • 推理加速：vLLM、TGI、TensorRT-LLM 等所有主流推理框架都已集成 FA2 作为默认 attention kernel。在 prefill 阶段尤其显著。
  • 长上下文可行性：FA2 + GQA 的组合使得 128K 上下文的 LLM 推理在单机上变得实用（见 Llama 3.1 的实现）。

算法层面的启示

  • 循环顺序的选择不是小事：FA2 中将内外循环的调换（以及随之而来的 warp 工作划分改变）是整个算法改进的核心。同样的计算量，不同的循环结构导致了 2x 的性能差距。
  • 尽可能延迟归一化：FA2 的 deferred normalization 思路在其他算法（如 FlashDecoding）中也被应用。只要能保证最终结果正确，中间状态保持"unnormalized"可以大幅减少运算量。
  • 非 matmul 运算是瓶颈：在现代 GPU 上，matmul 吞吐量远高于其他操作。任何频繁的逐元素操作（rescale、softmax）都需要被合并或减少到最低频率。
  • 硬件并行度决定算法设计：FA2 中序列维度并行的核心动机是"SM 数量 >> batch×heads"这个运行时观察。好的系统设计需要深入理解目标硬件的并行层次。
最重要的一个结论 Flash Attention 2 的改进不是靠更聪明的数学，而是靠更深入理解 GPU 硬件。FA1 到 FA2 的算法是等价的——产出完全相同的数值结果——但通过调整工作划分方式，将 GPU 利用率从 35% 提升到 72%。这说明对于 GPU kernel 工程，"如何并行"和"并行什么"同样重要，甚至更重要。
