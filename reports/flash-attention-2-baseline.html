1背景与动机

Transformer 模型的核心计算瓶颈是 自注意力机制（Self-Attention），其计算复杂度和内存占用均为 O(N²)，其中 N 是序列长度。随着 LLM 向更长上下文（16K、32K 乃至 1M tokens）演进，标准 Attention 的性能问题愈发突出。

核心问题
现代 GPU（如 A100 80GB）的计算能力（312 TFLOPS for BF16）增长速度远超 HBM 带宽（2 TB/s），导致标准 Attention 的瓶颈在于 内存 I/O 而非计算。对 N=2048、d=64 的标准 Attention 而言，HBM 读写量约是实际计算量所需带宽的 10× 以上，大量时间浪费在数据搬运上。

Flash Attention 1（2022, Dao et al.）通过 Tiling + Recomputation 技术将 Attention 从 HBM I/O 受限转为算术受限，取得了显著加速。然而 FA1 在以下方面仍有提升空间：

  • 存在不必要的 非矩阵乘法（non-matmul）FLOP，而这些运算在现代 GPU 上效率低下
  • 不支持 序列长度方向的并行化（批次大小小时 GPU 利用率低）
  • Warp 间的工作划分存在 通信开销（split-K 模式下需要在 shared memory 间同步）

2Flash Attention 1 核心回顾

理解 FA2 的改进，必须先清楚 FA1 做了什么。

2.1 标准 Attention 的内存问题

标准实现需要将中间矩阵 S = QKT（大小 N×N）和 P = softmax(S)（大小 N×N）物化到 HBM，然后再计算 O = PV。对于 N=2048、d=64，S 和 P 各需约 32MB（BF16），合计 64MB 的中间 HBM 写入/读取。

2.2 FA1 的 Tiling 算法

FA1 将 Q、K、V 分块载入 SRAM，通过 在线 Softmax（online softmax） 避免物化 N×N 矩阵：

  • 对 Q 的每个块，遍历所有 K、V 块（或 causal mask 允许的范围）
  • 用滑动更新的 running max (m) 和 running sum (l) 维持数值稳定的 softmax
  • 直接累积 O，最终无需 S、P 的 HBM 存储
O = softmax(QKT / √d) · V，FA1 无需物化中间 N×N 矩阵

2.3 FA1 的 Recomputation（重计算）

反向传播时不存储 N×N 的注意力权重矩阵，而是在需要时重新计算，以此换取内存节省。

FA1 的遗留问题
  • 非矩阵乘 FLOP 过多： FA1 的 forward pass 存在较多 rescaling（重缩放）操作，虽然 FLOP 数不多，但对 GPU 利用率影响显著——A100 矩阵乘单元（Tensor Core）的峰值吞吐是 FP32 标量单元的 16×，非矩阵乘运算实际消耗了比其 FLOP 数字所显示更多的时钟周期比例。
  • 并行度不足： FA1 只在 batch size 和 head 数上并行，当 batch size 或 head 数较小（长序列场景常见）时，SM 利用率低。
  • Warp 分工不理想： FA1 的 backward pass 中采用了 split-K 策略，不同 warp 处理不同的 K/V 块后需要同步，引入了额外的 shared memory 读写。

FA2 的三个核心改进方向：

  1. 减少非矩阵乘法 FLOP，提升 Tensor Core 利用率（通过重新设计 tiling 循环顺序实现）
  2. 沿序列长度方向并行化（引入跨 block 的 Q 分块并行，使得即使 batch=1 也能充分利用 GPU SM）
  3. 优化 Warp 间工作划分（减少 warp 间通信，降低 shared memory 读写开销）

4算法细节：重新设计的循环结构

4.1 FA1 vs FA2 的循环顺序对比

这是 FA2 最关键的算法改动之一。

FA1 Forward（简化）
for j in range(Tc): // 外循环: K,V 列块
load K_j, V_j
for i in range(Tr): // 内循环: Q 行块
load Q_i, O_i, l_i, m_i
S_ij = Q_i @ K_j.T
m̃_ij = rowmax(S_ij)
P̃_ij = exp(S_ij - m̃_ij)
l̃_ij = rowsum(P̃_ij)
// rescale O_i
m_i_new = max(m_i, m̃_ij)
l_i_new = e^(m_i-m_i_new)*l_i + e^(m̃_ij-m_i_new)*l̃_ij
O_i = (e^(m_i-m_i_new)*l_i*O_i + e^(m̃_ij-m_i_new)*P̃_ij@V_j) / l_i_new
write O_i, l_i, m_i back to HBM

注意：O_i 每次都需要从 HBM 读出、更新、再写回

FA2 Forward（简化）
for i in range(Tr): // 外循环: Q 行块
load Q_i
init O_i = 0, l_i = 0, m_i = -∞
for j in range(Tc): // 内循环: K,V 列块
load K_j, V_j
S_ij = Q_i @ K_j.T
m̃_ij = rowmax(S_ij)
P̃_ij = exp(S_ij - m̃_ij)
l̃_ij = rowsum(P̃_ij)
m_i_new = max(m_i, m̃_ij)
O_i = O_i * e^(m_i - m_i_new) + P̃_ij@V_j * e^(m̃_ij - m_i_new)
l_i += e^(m̃_ij - m_i_new) * l̃_ij + e^(m_i - m_i_new) * l_i
m_i = m_i_new
O_i /= l_i // 最终归一化（仅一次）
write O_i to HBM // 仅写一次！

关键：Q 块在寄存器/SRAM 中保持，O 只写一次

关键洞察：为什么交换循环顺序有效？

FA1 外循环是 K/V 分块（j），因此每次处理新 K/V 块时都需要更新并写回 O_i（因为不同 Q 块的 O 都可能被当前 K/V 块影响）。

FA2 外循环改为 Q 分块（i），对于固定的 Q_i，遍历完所有 K/V 块后 O_i 才完成，只需写回 HBM 一次。Q_i 和 O_i 在整个内循环期间保持在 SRAM（或寄存器）中，大幅减少 HBM I/O。

4.2 减少非矩阵乘 FLOP

FA1 在内循环每步都需要对 O_i 做 重缩放（rescale）：将 O_i 乘以 l_i_old / l_i_new，这是一个逐元素的向量乘法（非 matmul）。在 FA2 的新算法中：

  • 将归一化推迟到最后一步，中间累积的 O_i 是 未归一化版本（unnormalized）
  • 每次内循环只需做一次 rescale（乘以 exp 系数），最后统一除以 l_i
  • 虽然 FLOP 数量上变化不大，但消除了每次内循环中因 l 更新而导致的全量 rescale
为什么非矩阵乘 FLOP 昂贵？
在 NVIDIA A100 GPU 上：
  • BF16/FP16 矩阵乘（Tensor Core）：312 TFLOPS
  • FP32 标量/向量运算（CUDA Core）：19.5 TFLOPS
比率约为 16:1。即 1 FLOP 的 softmax exp 或向量 rescale，实际占用的时间相当于 16 FLOP 的矩阵乘。FA2 尽量将所有计算转化为矩阵乘，把非矩阵乘操作压缩到最少。

4.3 Causal Masking 优化

对于 causal（因果）attention，FA2 只在 Q 块与 K 块存在 diagonal 相交时才应用 masking。对于完全在对角线以下的块（全部有效），跳过 mask 操作，减少了约 15% 的计算量（统计 mask 判断分支的开销）。

5沿序列长度方向的并行化

5.1 FA1 的并行结构

FA1 的 CUDA kernel 在如下维度上并行：

  • Batch dimension (B)：每个 batch 样本独立
  • Head dimension (H)：每个 attention head 独立

因此 FA1 启动的 CUDA block 数量 = B × H。若 B=1, H=8（常见于推理场景），则只有 8 个 block 并行，无法充分利用 A100 的 108 个 SM。

低利用率场景
在 LLM 推理（batch=1）或长序列训练（序列长 4096，head 数少）场景下，FA1 的 SM 利用率可低至 <10%，成为严重瓶颈。

5.2 FA2 增加序列方向并行（Forward Pass）

FA2 将 Q 的分块也作为并行维度，Forward pass 的 block 数量变为：

CUDA Block 数 = B × H × ceil(N / Br)

其中 Br 是 Q 分块的行大小。对于 N=4096, Br=128，沿序列方向可产生 32 个额外并行单元，使总 block 数从 8 增至 256，极大提升 SM 利用率。

5.3 Backward Pass 的序列并行

Backward pass 需要计算 dQ, dK, dV。FA2 也将 K/V 分块的并行化引入 backward：

  • 外循环遍历 K/V 块，每个 K/V 块对应一个 CUDA block
  • 不同 K/V 块间独立计算 dK_j, dV_j，无需跨 block 同步
  • dQ 的计算需要 atomic add（不同 K/V 块都会贡献到同一 dQ 块），FA2 使用 FP32 的 atomic add 保证精度

6Warp 间工作划分优化

每个 CUDA block 内部由多个 warp（32 个线程的组） 构成。FA1 和 FA2 在 block 内的 warp 间工作划分策略存在显著差异。

6.1 FA1 的分割 K（Split-K）策略

FA1 在某些情况下采用 split-K 策略：将 K/V 矩阵在 K 维（head dimension）上分割给不同 warp 并行处理，最后 warp 间需要在 shared memory 中 reduce 中间结果。

Split-K 的问题
  • 需要通过 shared memory 在 warp 间同步 softmax 的统计量（max m, sum l）
  • shared memory 读写存在延迟（~20 cycles vs 寄存器的 ~4 cycles）
  • 同步点（__syncthreads）可能导致 warp 等待

6.2 FA2 的列方向分割（Split Along Sequence）

FA2 重新设计了 warp 间工作划分：

FA2 Warp 分工（示意）

FA2 选择将 Q 的行方向（序列维度）分给不同 warp，每个 warp 独立持有一部分 Q 行和对应的 O、m、l，遍历同一批 K/V 块。由于不同 warp 的 Q 行之间在 softmax 计算上完全独立，无需任何 warp 间通信，消除了 shared memory 同步开销。

FA2 Warp 划分的优势
  • 无 warp 间同步： 每个 warp 独立持有 m_i, l_i, O_i，无需 shared memory 中间结果
  • 寄存器复用率高： Q 块在内循环持续驻留在寄存器中，不被 evict
  • 更少的 shared memory bank conflict： K/V 的 shared memory layout 根据访问模式优化为 K 维度连续

7工程实现关键细节

7.1 Tile Size 选择

Tile size（分块大小 Br, Bc）的选择至关重要，需要平衡：

  • SRAM 容量约束： 每个 SM 的 shared memory 为 192KB（A100），需要容纳 Q_i（Br×d）、K_j（Bc×d）、V_j（Bc×d）及中间结果
  • 寄存器数量： O_i 若存于寄存器中（Br×d 个元素），则每线程寄存器消耗约为 Br×d / warp_size
  • occupancy（占用率）： 寄存器/shared memory 消耗越大，每 SM 同时运行的 warp 越少，隐藏延迟能力越差
典型配置（A100 BF16，d=64）： Br = 64 或 128，Bc = 64 或 128。FA2 paper 中报告对 d=64 使用 Br=Bc=128，对 d=128 使用 Br=Bc=64（因为更大的 d 消耗更多 shared memory）。

7.2 寄存器中的 O 累积

FA2 的一个关键实现选择：在内循环中将 Oi（Br×d 矩阵）保持在 寄存器文件 中而非 shared memory：

  • 寄存器访问延迟约 1–4 cycles，shared memory 约 20–30 cycles，HBM 约 600–800 cycles
  • 每线程持有 Oi 的一行（d 个元素），在内循环中直接累积
  • 代价是每个线程的寄存器压力增大，需要仔细调节 tile size 以维持足够 occupancy

7.3 Softmax 的 Log-Sum-Exp 技巧

FA2 存储 log-sum-exp（LSE） 而非分开存储 m 和 l：

LSEi = mi + log(li)

好处：在 backward pass 中，从 LSE 重建 softmax 只需一次 exp 运算；此外 LSE 合并减少了需要在内核间传递的中间变量数量（从 2 个减为 1 个），节省 HBM 带宽。

7.4 反向传播中的重计算

FA2 的 backward pass 沿用 FA1 的重计算策略，但有改进：

  • 不存储 N×N 的 attention weight P；在 backward 时利用存储的 LSE 和 Q/K/V 重新计算 P
  • 额外存储 O（forward 输出）和 LSE，共 O(N) 额外 HBM
  • backward 的 dP = dO · VT 可直接在 SRAM 中完成，不需要 HBM 中间落盘
// FA2 backward pass 伪代码（关键步骤） for j in range(Tc): // 外循环: K/V 块，并行 load K_j, V_j for i in range(Tr): load Q_i, O_i, dO_i, LSE_i // 重计算 P（利用 LSE 恢复 softmax） S_ij = Q_i @ K_j.T / sqrt(d) P_ij = exp(S_ij - LSE_i) // 利用已有 LSE，无需重算 max // 计算梯度 dV_j += P_ij.T @ dO_i dP_ij = dO_i @ V_j.T // D_i = rowsum(dO_i * O_i)（点乘求 softmax 梯度） D_i = rowsum(dO_i * O_i) dS_ij = P_ij * (dP_ij - D_i) dQ_i += dS_ij @ K_j / sqrt(d) // atomic add dK_j += dS_ij.T @ Q_i / sqrt(d) write dK_j, dV_j to HBM // dQ 通过 atomic add 归约（FP32 保证精度）

7.5 多查询注意力（Multi-Query / Grouped-Query Attention）支持

FA2 原生支持 Multi-Query Attention (MQA) 和 Grouped-Query Attention (GQA)：

  • MQA：所有 Q head 共享同一组 K/V head，K/V 块只需加载一次，显著减少 HBM 带宽
  • GQA：G 个 Q head 共享一组 K/V，FA2 将同组的 Q 合并处理，共享 K/V 的 SRAM 缓存
  • 实现上通过 kernel 参数控制 head 的 stride，无需代码分支

7.6 Head Dimension 的处理

FA1 只支持 d ∈ {16, 32, 64}，FA2 扩展支持了：

  • d = 128（GPT-4 等模型使用）：通过拆分为两个 d=64 的计算实现
  • 任意 d（非 2 的幂次）：通过 padding 到对齐边界处理

10局限性与后续工作

10.1 FA2 的局限

  • CUDA 专属：当时 FA2 仅提供 NVIDIA GPU CUDA 实现，不支持 AMD ROCm 或其他硬件（后续社区移植了 ROCm 版本）
  • 变长序列效率：不同序列长度 padding 浪费计算，FA2 提供了 variable-length API 但需要特殊处理
  • 特定 head dim 限制：虽然支持 d=128，但性能最优的实现仍集中在 d=64/128
  • 无法与 FlexAttention 兼容：任意 masking 模式（如 sliding window, block sparse）需要额外适配

10.2 后续衍生工作

Flash Attention 3（2024）
针对 H100 GPU 的 Hopper 架构优化：利用 TMA（Tensor Memory Accelerator）实现更高效的 async 数据搬运；利用 WGMMA（Warpgroup MMA）指令；实现 FP8 精度支持，forward 达到 H100 理论峰值的 75%。
FlexAttention（PyTorch）
基于 FA2 思路，通过 PyTorch Inductor 编译器自动生成针对任意 masking/scoring 函数的高效 Attention 内核，无需手写 CUDA。
Ring Attention
将 FA2 扩展到分布式设置：不同 device 持有不同序列段，通过 ring-allreduce 传递 K/V，实现百万 token 级序列长度的训练。
PagedAttention（vLLM）
将 FA2 的 tiling 思路扩展到推理服务的 KV Cache 管理，允许非连续内存存储 KV，大幅提升推理吞吐量。

11总结

Flash Attention 2 是一篇以工程实践驱动的算法优化为核心的论文。它没有改变 Attention 的数学含义，而是通过深刻理解 GPU 的计算层级（HBM / SRAM / 寄存器 / Tensor Core）来重新设计算法的执行顺序和并行策略。

三个核心贡献的本质
  1. 循环顺序翻转（Q 外循环）：让 O 只写 HBM 一次，减少 HBM 带宽浪费；同时减少不必要的非 matmul rescale 操作，让 Tensor Core 利用率最大化。
  2. 序列方向并行化：打破了 FA1 在 batch/head 小时的并发度瓶颈，使 GPU SM 在任何场景下都能满负荷运转——这对长序列 LLM 推理尤为关键。
  3. Warp 间工作划分优化：通过将 Q 行分配给不同 warp（而非 K 的特征维度），消除了 warp 间 softmax 统计量的 shared memory 同步，降低了 latency 并提升了寄存器复用。

FA2 的思想价值不仅在于对 Attention 本身的加速，更在于它提供了一个将硬件感知（hardware-aware）思维融入算法设计的范式：先问"数据在哪里？怎么搬运效率最高？"再问"计算顺序怎么排？"最后问"怎么把工作均匀分给所有计算单元？"。这一思路对 LLM 时代所有计算密集算子的优化都具有深远的指导意义。

一句话总结： FA2 通过交换 Q/K 的循环顺序减少 HBM 读写，通过增加序列维度并行提升 SM 利用率，通过改变 warp 内工作分配消除同步开销，三管齐下将 Attention 的 GPU 利用率从 ~50% 提升至 ~72%，速度较 FA1 提升约 2×，是 LLM 时代最重要的底层算子优化之一。
